# PartiPy

A Python suite for compatibility and partition analysis of aligned sequence data and binary 
rare genomic change data (such as transposable element insertion presence/absence data). 

The suite implements and extends ideas from two programs that are no longer publicly available:

- **reticulate** — Jakobsen & Easteal (1996) *CABIOS* 12:291–295
- **PartiMatrix** — Jakobsen, Wilson & Easteal (1997) *MBE* 14:474–484

Programs in the PartiPy suite implement the ability to generate **Lento Plots** similar to 
those generated by **PartiMatrix** (in svg format), output tsv files with the support and 
conflict values used in the Lento plots, and calculate compatibility scores. 

PartiPy programs are also designed to be used with phylogenomic datasets and allow the user 
to suppress graphical output and obtain large numbers of tsv files that can be used to generate 
summary statistics for large datasets.

The programs in the PartiPy suite have minimal dependencies (other than Python 3). 

---

## Programs

| Program | Purpose |
|---|---|
| `partcompat.py` | Compatibility score and Lento plot (with underlying compatibility data) for a single alignment |
| `id_compat.py` | Identify alignment sites compatible with specified bipartitions |
| `sample_loci.py` | Compatibility data and Lento plot for phylogenomic datasets by sampling sites from many loci |
| `analyze_loci.py` | Automate compatibility analyses for phylogenomic datasets |
| `lento_distances.py` | Compare individual-locus Lento spectra to a genome-wide centroid from `sample_loci.py` |
| `plot_lento.py` | Regenerate a LentoPlot SVG from an existing TSV (no reanalysis) |
| `compare_trees.py` | Compare internal branch lengths of two Newick trees or a Newick tree and compatibility data |
| `newick_to_biparts.py` | Extract bipartitions from a Newick tree and write in bipartition format (used by other PartiPy programs). Output hexadecimal indices of bipartitions |
| `paup_pstats.py` | Parsimony statistics (CI, RI, RC) via PAUP\* for binary matrices |

All programs except `paup_pstats.py` require only Python 3 (standard library 
only — no third-party packages).  `paup_pstats.py` additionally requires 
PAUP\* (Swofford 2002), which must be installed separately.

## partcompat.py

### What it does

Given a DNA multiple-sequence alignment or binary data matrix, `partcompat.py`:

1. Identifies parsimony-informative (PI) sites under the chosen coding scheme (RY-coding is 
the default for DNA sequencies).
2. Computes the **compatibility score**: the proportion of all pairs of parsimony-informative 
binary characters that are pairwise compatible under the Estabrook–McMorris criterion
3. For each unique bipartition (split) supported by one or more PI sites, computes raw support 
and conflict counts, then applies Lento normalization so that total support equals total 
conflict across all bipartitions.
4. Writes a **TSV** table of bipartition support and conflict values.
5. Writes **LentoPlot SVG** file(s): support bars pointing up (green), conflict bars pointing 
down (red), one column per bipartition, with a dot schematic showing taxon groupings below.

There are a number of ways to control the details of the **LentoPlot SVG** file formatting 
that facilitate the production of high-quality figures. The output of the svg files can also 
be suppressed for batch analyses of datasets.

### Input formats

Auto-detected from file content:

- FASTA (`.fasta`, `.fa`, etc.)
- Relaxed PHYLIP
- Nexus

Both nucleotide and binary (0/1) data are accepted.  Binary data is detected automatically; 
the `--mode` setting is ignored for binary data.

### Site coding modes

The `--mode` option (case-insensitive; reversed pairs accepted, e.g. `YR` equals `RY`) controls 
how nucleotide columns are converted to binary characters:

| Mode | Group 0 | Group 1 | IUPAC resolved | IUPAC → missing |
|---|---|---|---|---|
| `RY` *(default)* | A, G (purines) | C, T (pyrimidines) | R→0, Y→1 | S, W, K, M |
| `KM` | A, C (amino) | G, T (keto) | M→0, K→1 | R, Y, S, W |
| `WS` | A, T (weak) | C, G (strong) | W→0, S→1 | R, Y, K, M |
| `perfectRY` | A, G (purines) | C, T (pyrimidines) | none | all IUPAC |
| `perfectKM` | A, C (amino) | G, T (keto) | none | all IUPAC |
| `perfectWS` | A, T (weak) | C, G (strong) | none | all IUPAC |
| `transitions` | A↔G or C↔T (axis inferred per column) | *(same axis)* | R or Y within-axis →0.5 | all cross-axis IUPAC |
| `transitionsAG` | A | G | R→missing | all cross-axis bases/IUPAC |
| `transitionsCT` | C | T | Y→missing | all cross-axis bases/IUPAC |
| `2state` | whichever 2 bases appear | — | none | all IUPAC |
| `partimatrix` | column-dependent | — | none | all two-state IUPAC |

The `perfect` variants additionally require that each column contains **exactly two distinct 
unambiguous nucleotide states** (A, C, G, or T only — no IUPAC ambiguity codes and no gaps), 
and that the two states fall on **opposite sides** of the coding axis (e.g. one purine and 
one pyrimidine for `perfectRY`). This matches the *perfect transversions* criterion of Tiley 
et al. (2020): a parsimony-informative site where both the binary-coded character and the 
underlying nucleotide identity are unambiguous. Sites with any missing data, gaps, or IUPAC 
codes are excluded.  The `perfect` criterion reduces the number of informative sites but it 
has the potential to restrict consideration to less homoplastic characters (e.g., sites that 
cannot accumulate within-group nucleotide substitutions, such A↔G transitions within the purine group).

Reversed pairs are accepted case-insensitively for all modes: `perfectYR`, `perfectMK`, and 
`perfectSW` are equivalent to `perfectRY`, `perfectKM`, and `perfectWS` respectively.

The `transitions` modes restrict analysis to **transition-only** sites, complementing the 
transversion-based RY/KM/WS and perfect modes.  `transitions` (alias `transition`) infers the 
relevant axis (A↔G or C↔T) per column from the unambiguous bases present; any character that 
definitively indicates a transversion (including IUPAC codes spanning both axes) excludes the 
site entirely.  `transitionsAG`/`transitionAG` (and reversed `transitionsGA`/`transitionGA`) and 
`transitionsCT`/`transitionCT` (and reversed `transitionsTC`/`transitionTC`) restrict to a single 
axis.  Within the relevant axis, the ambiguity code R (for AG columns) or Y (for CT columns) is 
treated as missing data, contributing the standard +0.5 to each bipartition resolution when it is 
the single missing taxon.  These modes are useful for exploring how varying levels of homoplasy 
(transitions are typically more homoplastic than transversions) affect the compatibility score and 
Lento plot — for example, to separate the contribution of homoplasy from hemiplasy in datasets 
where incomplete lineage sorting is suspected.

In `partimatrix` mode, columns with exactly two distinct unambiguous bases are treated as 
binary regardless of which bases they are; columns with three or four bases are RY-collapsed 
before being treated as binary.  This replicates the historical behaviour of the original 
**PartiMatrix** program.

Gaps (`-`), `N`, and `?` are always treated as missing data.  A site with two or more missing 
characters is excluded entirely.  A site with exactly one missing character contributes +0.5 
to each of the two bipartitions that would result from assigning the missing taxon to either group.

### Canonical bipartition representation

Bipartitions are stored as integer bitmasks with the following conventions:

- Bit string is read left-to-right = first taxon to last taxon in the taxon ordering.
- The **canonical form** always has the first bit set to 1.  If the raw pattern starts with 
0, all bits are flipped (the complement represents the same split).
- The **Jakobsen hex** code reported in TSV output is the bitwise complement of the canonical 
bitmask expressed as uppercase hexadecimal, matching the bipartition indexing used by the 
original programs.

### Usage

```
partcompat.py -i INFILE -o PREFIX [options]
```

**Required:**

| Flag | Description |
|---|---|
| `-i` / `--infile` | Input alignment (FASTA, relaxed PHYLIP, or Nexus) |
| `-o` / `--outfile` | Output file prefix; produces `PREFIX.tsv` and `PREFIX.svg` |

**Site coding:**

| Flag | Description |
|---|---|
| `-m` / `--mode` | Coding mode: `RY`, `KM`, `WS`, `perfectRY`, `perfectKM`, `perfectWS`, `transitions`, `transitionsAG`, `transitionsCT`, `2state`, `partimatrix` (and reversed pairs, e.g. `YR`, `perfectYR`, `transitionGA`); default: `RY` |
| `--codon-positions` | Restrict to `1`, `2`, `3`, `12`, `13`, `23`, or `123` (default: `all`) |

**Taxon ordering:**

| Flag | Description |
|---|---|
| `--tax-order FILE` | One taxon name per line; sets bit-string position order.  Sequences present in the alignment but absent from the file are appended with a warning.  Strongly recommended when comparing outputs across datasets. |

**Bipartition files** (all use the same plain-text format: one bipartition per line as a binary string or Jakobsen hex string; blank lines and lines starting with `#` are ignored):

| Flag | Description |
|---|---|
| `-t` / `--tree-biparts FILE` | Bipartitions from a reference tree.  Matching columns in the LentoPlot are drawn as **outline** bars rather than solid bars, making it easy to see which tree splits are supported vs. conflicted. |
| `-p` / `--plot-biparts FILE` | Display filter: only bipartitions in this file appear in the SVG and TSV.  All computation (compatibility score, normalization) uses the full data.  Bipartitions with zero support in the data are still plotted (conflict-only bars), which lets users visualise the absence of support for specific splits.  Duplicate entries are silently deduplicated. |
| `-b` / `--bipart-order FILE` | Column ordering: bipartitions listed in this file appear as the leftmost columns (in file order); remaining columns follow sorted by support − conflict.  Duplicate entries trigger an error.  This allows direct visual comparison of LentoPlots from different datasets. |

**Output control:**

| Flag | Description |
|---|---|
| `--no-svg` | Suppress SVG output (useful for batch runs) |
| `--score-file` | Write compatibility score to `PREFIX.compat.txt` (one line) |
| `--truncate N` | Write a second SVG LentoPlot to `PREFIX.trunc.svg` that only shows the first N columns|
| `--no-hex` | Suppress Jakobsen hex labels above the LentoPlot |

When `-t` / `--tree-biparts` is supplied, the following are also reported:

- **stdout**: the 1-based column position of the last (rightmost) tree bipartition in the full Lento plot, with its binary string and Jakobsen hex.  This helps the user choose a sensible value for `--truncate` to capture all tree bipartitions.
- **`PREFIX.tree_bipart.tsv`**: a separate TSV restricted to tree bipartitions, sorted by support − conflict (same ordering as the main TSV), with an extra `plot_position` column giving each bipartition's 1-based column index in the full plot (empty for tree bipartitions absent from the data).

### Outputs

**`PREFIX.tsv`** — tab-separated bipartition table, sorted by
support_norm − conflict\_norm (descending), or by `--bipart-order` if
supplied.

| Column | Description |
|---|---|
| `bipartition_binary` | Canonical binary string, tick-prefixed (`'`) to prevent spreadsheet misinterpretation |
| `jakobsen_hex` | Jakobsen hexadecimal code (complement convention) |
| `support_raw` | Raw support weight (number of PI sites matching this split) |
| `conflict_raw` | Raw conflict weight (number of PI sites four-gamete-incompatible with this split) |
| `support_norm` | Lento-normalized support |
| `conflict_norm` | Lento-normalized conflict (scaled so Σconflict = Σsupport) |
| `grouping` | Human-readable split: `(taxon,…)|(taxon,…)` |

**`PREFIX.svg`** — LentoPlot vector graphic.  Imports directly into various SVG-capable tools for publication-quality editing.

**`PREFIX.trunc.svg`** (with `--truncate N`) — LentoPlot vector graphic limited to the first N of the Lento plot (same ordering as the full plot in `PREFIX.svg`).

**`PREFIX.tree_bipart.tsv`** (with `-t` / `--tree-biparts`) — tab-separated table restricted to the tree bipartitions, sorted by support_norm − conflict_norm.  Columns are the same as `PREFIX.tsv` plus `plot_position` (1-based column index in the full Lento plot; empty if the bipartition is absent from the data).  Bipartitions present in the tree file but absent from the data appear with zero support/conflict and no plot position.

**`PREFIX.compat.txt`** (with `--score-file`) — single line containing the compatibility score as a decimal fraction.

**stdout** — compatibility score with pair counts, e.g.:
```
Compatibility score: 0.823456  (4521 compatible pairs / 5050 total pairs)
```

### Examples

Basic run:
```bash
python3 partcompat.py -i alignment.fasta -o results
```

With taxon ordering and tree bipartitions highlighted:
```bash
python3 partcompat.py -i alignment.fasta -o results \
    --tax-order taxa.txt \
    -t tree.biparts
```

Restrict plot to tree bipartitions, with no hex labels:
```bash
python3 partcompat.py -i alignment.fasta -o results \
    --tax-order taxa.txt \
    -t tree1.biparts \
    -p tree1.biparts \
    --no-hex
```

Use truncation to produce a publication-ready figure limited to tree bipartitions:
```bash
python3 partcompat.py -i alignment.fasta -o results \
    --tax-order taxa.txt \
    -t tree.biparts \
    -p tree.biparts \
    --truncate 32
```
The stdout line "last column = 32" tells you the last tree bipartition falls at
column 32 in the full plot, so `--truncate 32` (or slightly higher) captures
all tree bipartitions in `results.trunc.svg`.

Compare two trees side by side (same column order):
```bash
# Generate a common ordering from tree 1
python3 newick_to_biparts.py tree1.nwk taxa.txt > tree1.biparts

# Dataset A
python3 partcompat.py -i alignA.fasta -o resultsA \
    --tax-order taxa.txt -b tree1.biparts

# Dataset B (identical column order)
python3 partcompat.py -i alignB.fasta -o resultsB \
    --tax-order taxa.txt -b tree1.biparts
```

## id_compat.py

Identifies the alignment sites (under a chosen coding scheme) that support
one or more user-specified bipartitions, either fully (no missing data) or
partially (exactly one missing taxon, the standard +0.5 case used throughout
the suite).  Useful for examining the specific site patterns -- and, with an
annotation file, their genomic positions or other metadata -- that underlie
support for particular branches.

### Usage

```
id_compat.py -i ALIGNMENT --tax-order FILE -o PREFIX \
    (--bipart BIPART [--bipart BIPART ...] | --bipart-file FILE) [options]
```

**Required:**

| Flag | Description |
|---|---|
| `-i` / `--infile` | Input alignment (FASTA, relaxed PHYLIP, or Nexus) |
| `--tax-order FILE` | Taxon order file (one name per line) |
| `-o` / `--outfile` | Output prefix; produces `PREFIX.txt` (and `PREFIX.fasta` with `--fasta-output`) |

**Bipartition specification (at least one required):**

| Flag | Description |
|---|---|
| `--bipart BIPART` | A single bipartition, as a binary string or Jakobsen hex.  May be repeated for multiple bipartitions. |
| `--bipart-file FILE` | File listing bipartitions (binary or hex, one per line, `#` comments allowed; same format as `-t`/`--tree-biparts` elsewhere in the suite) |

**Other options:**

| Flag | Description |
|---|---|
| `-m` / `--mode` | Site coding mode (default: RY).  Same options as `partcompat.py`, including `perfectRY`/`perfectKM`/`perfectWS` and `transitions`/`transitionsAG`/`transitionsCT`. |
| `--codon-positions` | Restrict to specific codon positions (default: all) |
| `--annotation FILE` | Per-character annotations (see below) |
| `--verbose-outfile` | Include taxon groupings `(tax1,tax2,...|tax3,tax4,...)` for each bipartition |
| `--fasta-output {binary,original}` | Write `PREFIX.fasta` containing the union of all sites supporting any queried bipartition, as recoded binary characters or original nucleotides |

### Annotation file formats

Two formats are auto-detected:

1. **One annotation per line**, with exactly as many lines as the alignment
   has characters.  Line *n* annotates alignment position *n* (1-based).
2. **Two tab-separated columns**: `<1-based position>\t<annotation>`.
   Positions not listed are labeled `unannotated`.

### Output

**`PREFIX.txt`** begins with the taxon list in analysis order, then one block
per queried bipartition:

```
=== Bipartition: 110000  (hex: F) ===
(tax1,tax2)|(tax3,tax4,tax5,tax6)        <- only with --verbose-outfile
Full support (no missing): 5, 10, 27, 53
Partial support (1 missing): 12, 44
5    <annotation>
10   <annotation>
12*  <annotation>
27   <annotation>
44*  <annotation>
53   <annotation>
```

Site positions are 1-based alignment columns.  When an annotation file is
supplied, every full- or partial-support position is listed with its
annotation; partial-support positions are marked with a trailing `*`.

**`PREFIX.fasta`** (with `--fasta-output`) contains the union of all sites
(full- or partial-support, across all queried bipartitions), in alignment
order, for every taxon in `--tax-order`.  With `--fasta-output original`,
the original alignment characters are written.  With `--fasta-output binary`,
the recoded 0/1/`?` characters (under the chosen `--mode`) are written, with
`?` for the missing taxon at partial-support sites.  This file is intended
for examining the complete character set supporting a set of bipartitions --
for example, the bipartitions of a candidate tree that maximize compatible
characters (the maximum compatibility problem).

## sample_loci.py

### What it does

`sample_loci.py` implements a phylogenomic resampling framework for compatibility analysis.  
For each replicate it:

1. Scans a directory of alignment files.
2. Randomly selects one parsimony-informative site from each eligible locus.
3. Assembles the selected sites into a binary relaxed PHYLIP file.
4. Calls `partcompat.py` as a subprocess to produce a TSV and compatibility score for that replicate.

After all replicates, it summarises the results:

- Mean (or median with `--robust`) and SD (or scaled MAD) of support and conflict per bipartition across replicates.
- Mean ± SD of the compatibility score.
- A summary LentoPlot SVG and TSV.

The intended use is comparison of within-locus compatibility (run `partcompat.py` directly 
on individual locus alignments) or compatibility for binary rare genomic change data to genome-wide 
compatibility (which can be assessed by running `sample_loci.py` on a directory with alignment 
files for many loci).  Because unlinked loci from a genome-wide sample are subject to 
gene-tree/species-tree discordance, the genome-wide compatibility score is generally expected 
to be lower than within-locus scores.

The `--variable-sites` option addresses the information-content confound in such comparisons: 
individual loci vary widely in the number of parsimony-informative (PI) sites they contain, and 
loci with fewer PI sites tend to have higher variance in Lento distances from the genome-wide 
centroid regardless of any biological signal.  By drawing a variable number of sites per replicate 
from a distribution matching the observed range of individual-locus PI-site counts (e.g. 
`--variable-sites uniform:47:392`), the replicate distribution more closely matches the variance 
expected from information content alone, providing a better null distribution against which to 
assess individual loci.

### Usage

```
sample_loci.py -i INDIR -o OUTDIR --tax-order FILE [options]
```

**Required:**

| Flag | Description |
|---|---|
| `-i` / `--indir` | Directory containing alignment files (FASTA, PHYLIP, or Nexus; auto-detected) |
| `-o` / `--outdir` | Output directory (created if absent) |
| `--tax-order FILE` | Taxon order file (required; same file used for `partcompat.py`) |

**Sampling:**

| Flag | Description |
|---|---|
| `-n` / `--replicates N` | Number of replicates (default: 100) |
| `--seed N` | Random seed for reproducibility.  If omitted, a seed is generated and reported in `run_info.txt`. |
| `--boot-locus` | Enable locus bootstrapping: loci are resampled with replacement (multinomial) rather than each contributing exactly one site per replicate.  Useful for estimating variance attributable to locus sampling. |
| `--variable-sites DIST` | Draw a variable number of sites N per replicate from the distribution DIST rather than one site per locus.  Formats: `uniform:MIN:MAX` (draw N uniformly from [MIN, MAX]) or `fixed:N` (always use N).  Without `--downsample`, N sites are drawn i.i.d. from the locus pool and N may exceed the number of loci.  Compatible with `--boot-locus` (loci are reweighted before drawing). |
| `--downsample` | Used with `--variable-sites`: select N loci **without replacement** (N must be ≤ n_eligible) and draw one site per locus.  Incompatible with `--boot-locus`.  The maximum value of DIST must not exceed the number of eligible loci. |
| `--max-missing-taxa FRAC` | Skip a locus if more than this fraction of taxa from `--tax-order` are absent from the alignment file (0.0–1.0, default: 0.5). |

**Site coding** (same options as `partcompat.py`):

| Flag | Description |
|---|---|
| `-m` / `--mode` | Coding mode (default: `RY`) |
| `--codon-positions` | Restrict to specific codon positions (default: `all`) |

**Bipartition files** (passed through to each `partcompat.py` subprocess call and applied to summary outputs):

| Flag | Description |
|---|---|
| `-t` / `--tree-biparts FILE` | Reference tree bipartitions (outline bars) |
| `-p` / `--plot-biparts FILE` | Display filter |
| `-b` / `--bipart-order FILE` | Column ordering |

**Summary statistics:**

| Flag | Description |
|---|---|
| `--robust` | Report median and scaled MAD (×1.4826) instead of mean and SD in the summary TSV variability columns |

**Output control:**

| Flag | Description |
|---|---|
| `--no-hex` | Suppress hex labels in all SVG outputs |
| `--redo` | Overwrite existing output directory contents (exits with an error if replicate files exist and `--redo` is not supplied) |

### Output directory

```
replicate_NNNN.phy        binary relaxed PHYLIP (one per replicate)
replicate_NNNN.tsv        bipartition table from partcompat.py
replicate_NNNN.compat.txt compatibility score (one line)
summary.tsv               mean/median support and conflict per bipartition
summary.svg               LentoPlot of averaged bipartition data
summary.compat.txt        mean ± SD of compatibility scores across replicates
run_info.txt              seed, parameters, eligible and skipped loci
```

### Summary TSV columns

The summary TSV has the same columns as a standard `partcompat.py` TSV, plus two variability 
columns appended before `grouping`:

| Column | Description |
|---|---|
| `support_sd` | SD of `support_raw` across replicates (or `support_mad` with `--robust`) |
| `conflict_sd` | SD of `conflict_raw` across replicates (or `conflict_mad` with `--robust`) |

Variability columns report **raw** (non-normalized) values, on the same scale as `support_raw` 
and `conflict_raw`.  The normalized columns (`support_norm`, `conflict_norm`) reflect means 
(or medians) of raw values normalized after averaging, which is the statistically appropriate 
order of operations.

### Missing taxa

Taxa present in `--tax-order` but absent from a locus alignment are padded with `?` (missing 
data) before site selection.  The per-site missing-data rules of `partcompat.py` then apply: 
sites with two or more missing characters are excluded from consideration; sites with exactly 
one missing character contribute +0.5 to each of the two possible bipartition resolutions.

The `--max-missing-taxa` threshold is evaluated at the locus level (before site selection) based on how many taxa from `--tax-order` are absent from the alignment file entirely.

### Example

```bash
# Generate bipartitions from a reference tree
python3 newick_to_biparts.py ref_tree.nwk taxa.txt > ref.biparts

# Genome-wide compatibility (100 replicates, locus bootstrap)
python3 sample_loci.py \
    -i loci_directory/ \
    -o genome_wide_results/ \
    --tax-order taxa.txt \
    -n 100 \
    --boot-locus \
    -t ref.biparts \
    -p ref.biparts \
    --seed 12345

# Variable information content: match the PI-site range of individual loci
# (useful for controlling for information-content effects in lento_distances.py)
python3 sample_loci.py \
    -i loci_directory/ \
    -o genome_wide_variable/ \
    --tax-order taxa.txt \
    -n 1000 \
    --variable-sites uniform:47:392 \
    -p ref.biparts \
    --seed 12345

# Downsampling: fewer loci, one site each (e.g. for comparison at smaller scales)
python3 sample_loci.py \
    -i loci_directory/ \
    -o genome_wide_down/ \
    --tax-order taxa.txt \
    -n 100 \
    --variable-sites uniform:20:80 --downsample \
    -p ref.biparts \
    --seed 12345

# Individual locus analysis (shell loop)
mkdir locus_results
for f in loci_directory/*.fasta; do
    base=$(basename "$f" .fasta)
    python3 partcompat.py \
        -i "$f" \
        -o "locus_results/$base" \
        --tax-order taxa.txt \
        -t ref.biparts \
        -p ref.biparts \
        --no-svg \
        --score-file
done
```

## analyze_loci.py

Runs `partcompat.py` on every alignment file in a directory, producing one
TSV (and compatibility score file) per locus in a flat output directory --
without any SVG output.  This is the per-locus counterpart to
`sample_loci.py`: where `sample_loci.py` samples one site per locus across
many replicates to characterize genome-wide signal, `analyze_loci.py` runs
the full compatibility analysis on each locus individually, so that
individual-locus results can be compared to the genome-wide distribution
(e.g. with `lento_distances.py`).

### Usage

```
analyze_loci.py -i INDIR -o OUTDIR --tax-order FILE [options]
```

**Required:**

| Flag | Description |
|---|---|
| `-i` / `--indir` | Input directory containing alignment files (FASTA, PHYLIP, or Nexus; auto-detected) |
| `-o` / `--outdir` | Output directory for TSV files (created if absent) |
| `--tax-order FILE` | Taxon order file (one name per line); required |

**Site coding** (same options as `partcompat.py`):

| Flag | Description |
|---|---|
| `-m` / `--mode` | Coding mode (default: `RY`) |
| `--codon-positions` | Restrict to specific codon positions (default: `all`) |

**Bipartition files** (passed through to each `partcompat.py` call; should match the files used with `sample_loci.py` for comparable results):

| Flag | Description |
|---|---|
| `-p` / `--plot-biparts FILE` | Display filter -- must match the file used with `sample_loci.py` so that TSVs from both programs cover the same bipartition set |
| `-t` / `--tree-biparts FILE` | Reference tree bipartitions (recorded in the TSV grouping column) |
| `-b` / `--bipart-order FILE` | Bipartition column ordering |

**Eligibility and output control:**

| Flag | Description |
|---|---|
| `--max-missing-taxa FRAC` | Skip a locus if this fraction of taxa from `--tax-order` are absent (default: 0.5) |
| `--no-hex` | Suppress Jakobsen hex labels (passed to `partcompat.py`) |
| `--redo` | Overwrite existing output directory contents |

### Output directory

```
LOCUS.tsv          bipartition table from partcompat.py (one per input alignment)
LOCUS.compat.txt   compatibility score (two lines: score, n_pi_sites)
run_info.txt       parameters, eligible/skipped/failed loci
```

Loci are skipped (not processed) if they have too many missing taxa or no
parsimony-informative sites under the chosen scheme; skipped and failed loci
are listed by name in `run_info.txt`.

### Example

```bash
python3 analyze_loci.py \
    -i loci_directory/ \
    -o gene_results/ \
    --tax-order taxa.txt \
    -p ref.biparts \
    --mode RY
```

## lento_distances.py

Computes distance metrics between individual-locus Lento spectra (from
`analyze_loci.py`) and the genome-wide centroid spectrum (the `summary.tsv`
produced by `sample_loci.py`), and between each replicate spectrum and that
same centroid.  This quantifies how unusual each individual locus's
support/conflict pattern is relative to the genome-wide average, and how that
compares to the sampling variation already present among `sample_loci.py`
replicates.

### Usage

```
lento_distances.py -s SAMPLE_DIR -g GENE_DIR -o PREFIX [options]
```

**Required:**

| Flag | Description |
|---|---|
| `-s` / `--sample-dir DIR` | `sample_loci.py` output directory (contains `summary.tsv` and `replicate_*.tsv`) |
| `-g` / `--gene-dir DIR` | `analyze_loci.py` output directory (contains one TSV per gene) |
| `-o` / `--outfile PREFIX` | Output prefix; produces `PREFIX.tsv` and `PREFIX.svg` |

**Options:**

| Flag | Description |
|---|---|
| `--distances LIST` | Comma-separated distance metrics to compute: `euclidean`, `manhattan`, `cosine`, or `all` (default: `all`) |
| `--vectors {support,both}` | Vector composition: `support` uses `support_norm` only; `both` concatenates `support_norm` and `conflict_norm` (default: `both`) |
| `--no-svg` | Suppress SVG output |

Both input directories must have been produced using the **same**
`--tax-order`, `--mode`, and `--plot-biparts` settings, so that bipartition
sets are directly comparable across the centroid, replicates, and individual
loci.

### How it works

1. Reads the bipartition set and `support_norm`/`conflict_norm` values from
   `summary.tsv` (the centroid).
2. Reads each `replicate_*.tsv` and each gene TSV in `--gene-dir`, building a
   vector of `support_norm` (and optionally `conflict_norm`) values over the
   centroid's bipartition set.
3. Sum-to-one normalizes each vector (so the support values across
   bipartitions sum to 1) before computing distances, removing the effect of
   differing total signal strength.
4. Computes the requested distance(s) between each replicate/gene vector and
   the centroid vector.

A warning is printed for any replicate or gene TSV whose bipartition set
differs substantially from the centroid's (more than 50% symmetric
difference), which usually indicates `--plot-biparts` was not applied
consistently between `sample_loci.py` and `analyze_loci.py`.

### Outputs

**`PREFIX.tsv`** -- one row per replicate and per gene, with columns:

| Column | Description |
|---|---|
| `dataset` | Replicate or gene identifier |
| `type` | `replicate` or `gene` |
| `n_biparts` | Number of bipartitions with nonzero support in this dataset |
| `sum_support_raw` | Sum of `support_raw` across all bipartitions (a measure of information content / number of PI sites) |
| `compat_score` | Compatibility score from the corresponding `.compat.txt` file, if present |
| `<distance>_<vector>` | One column per requested distance/vector combination, e.g. `euclidean_support`, `cosine_both` |

A footer block reports mean and SD of each distance column, separately for
replicates and genes.

**`PREFIX.svg`** -- strip plot: one row of jittered points per distance
metric, replicates in blue and genes in orange, with mean ± SD bars
overlaid for each group.  This makes it straightforward to see whether
individual loci are more variable than the replicate-sampling distribution
alone would predict.

### Interpreting `sum_support_raw`

`sum_support_raw` is a proxy for information content: loci with few
parsimony-informative sites have noisier Lento spectra and therefore tend to
sit further from the centroid by chance alone, independent of any biological
signal.  Plotting a distance column against `sum_support_raw` (e.g. in a
spreadsheet) can help distinguish loci that are unusual because of low
information content from loci that are unusual despite having ample signal.
The `--variable-sites` option of `sample_loci.py` (see above) was added
specifically to give the replicate distribution a comparable range of
information content, making this comparison more meaningful.

### Example

```bash
python3 lento_distances.py \
    -s genome_wide_results/ \
    -g gene_results/ \
    -o distances/comparison
```

## plot_lento.py

Regenerates a LentoPlot SVG from an existing TSV file (from `partcompat.py`,
`sample_loci.py`'s `summary.tsv`, or `analyze_loci.py`) without rerunning any
analysis.  The normalized `support_norm` and `conflict_norm` values already
present in the TSV are used directly, preserving whatever normalization was
applied during the original analysis -- the goal is figure control (column
ordering, truncation, omitted bipartitions, hex labels), not reanalysis.

This is the natural tool for iterating on figure appearance: rerun
`plot_lento.py` with different `-t`/`-p`/`-b`/`--omit-biparts`/`--truncate`
options as many times as needed, without repeating the (potentially slow)
compatibility computation.

### Usage

```
plot_lento.py -i INFILE --tax-order FILE -o PREFIX [options]
```

**Required:**

| Flag | Description |
|---|---|
| `-i` / `--infile FILE` | Input TSV from `partcompat.py`, `sample_loci.py` (`summary.tsv`), or `analyze_loci.py` |
| `--tax-order FILE` | Taxon order file; must match the file used to produce the TSV (validated -- see below) |
| `-o` / `--outfile PREFIX` | Output prefix; produces `PREFIX.svg` (and `PREFIX.trunc.svg` with `--truncate`) |

**Bipartition files** (same format as `partcompat.py`):

| Flag | Description |
|---|---|
| `-t` / `--tree-biparts FILE` | Reference tree bipartitions (outline bars); also reports the column position of the last tree bipartition, as in `partcompat.py` |
| `-p` / `--plot-biparts FILE` | Display filter: only these bipartitions appear in the SVG |
| `-b` / `--bipart-order FILE` | Column ordering file |
| `--omit-biparts FILE` | Bipartitions to exclude from the SVG (useful for omitting outgroup branches) |

**Output control:**

| Flag | Description |
|---|---|
| `--truncate N` | Also write `PREFIX.trunc.svg` with only the first N columns |
| `--no-hex` | Suppress Jakobsen hex labels above the plot |

### Taxon order validation

Because `plot_lento.py` reconstructs the dot schematic and taxon labels from
the TSV without re-reading any alignment, it performs two checks against
`--tax-order`:

1. **Bit-string length** -- every `bipartition_binary` value in the TSV must
   have the same length as `--tax-order`.  A mismatch is a hard error listing
   the offending bipartitions, since it usually means the wrong taxon order
   file was supplied.
2. **Taxon names** -- every taxon name appearing in the TSV's `grouping`
   column must be present in `--tax-order` (hard error if not, since the
   grouping column is the ground truth for what taxa were used).  Taxa present
   in `--tax-order` but absent from any grouping are reported as a NOTE
   (expected if those taxa never appear on the smaller side of any
   non-trivial bipartition in this dataset).

### Example

```bash
python3 plot_lento.py \
    -i results.tsv \
    --tax-order taxa.txt \
    -o results_fig \
    -t tree.biparts \
    -p tree.biparts \
    --truncate 24
```

## compare_trees.py

Compares internal branch lengths between two Newick trees, or compares a
tree against Lento-plot support values from `partcompat.py` or
`sample_loci.py`.  Branches are matched by bipartition (canonical bitmask).
Branches present in only one dataset have value 0 for the absent side.
The output of this program can be used to compare support and conflict in
the Lento plots to branch lengths in trees inferred using other methods.

### Modes

**Tree-vs-tree** (`-1` and `-2`): compare internal branch lengths between two
Newick trees.  Useful for comparing timetree branch lengths against coalescent
branch lengths, or comparing alternative tree topologies.

**Tree-vs-Lento** (`-1` and `-l`): compare tree branch lengths against
`support_norm` values from a Lento TSV.  The Lento support values replace
tree 2 branch lengths.  Bipartitions with Lento support but absent from the
tree appear at the right with a zero tree-1 bar.

### Usage

```
compare_trees.py -1 TREE1.nwk (-2 TREE2.nwk | -l LENTO.tsv) --tax-order FILE -o PREFIX
```

**Required:**

| Flag | Description |
|---|---|
| `-1` / `--tree1` | First Newick tree file |
| `-2` / `--tree2` | Second Newick tree file *(tree-vs-tree mode)* |
| `-l` / `--lento` | Lento TSV from `partcompat.py` or `sample_loci.py` *(tree-vs-Lento mode; `support_norm` used as branch lengths)* |
| `--tax-order FILE` | Taxon order file; must match the file used with `partcompat.py` |
| `-o` / `--outfile` | Output prefix; produces `PREFIX.tsv` and `PREFIX.svg` |

**Plot options:**

| Flag | Description |
|---|---|
| `--no-normalize` | Do not scale values to equal sums before plotting (default: normalize so both totals are equal) |
| `--no-hex` | Suppress Jakobsen hex labels above the plot |
| `--truncate N` | Also write `PREFIX.trunc.svg` showing only the first N pairs |
| `--omit-biparts FILE` | File of bipartitions to exclude from the SVG only (same binary/hex format as `--tree-biparts`; TSV is unaffected) |
| `--no-svg` | Suppress SVG output; TSV is always written |

### SVG output

The SVG shows paired upward bars — one per bipartition — with tree 1 (blue)
on the left and tree 2 / Lento support (orange) on the right of each pair,
with a gap separating consecutive pairs.  By default both datasets are scaled
so their totals are equal, making bar heights directly comparable.  The dot
schematic and Jakobsen hex labels follow the same conventions as the LentoPlot.

### Output TSV columns

| Column | Description |
|---|---|
| `bipartition_binary` | Canonical binary string (tick-prefixed) |
| `jakobsen_hex` | Jakobsen hex code for cross-reference with LentoPlot output |
| `length_tree1` | Internal branch length in tree 1 (0.0 if branch absent) |
| `length_tree2` or `support_norm` | Branch length / Lento support for dataset 2 (0.0 if absent) |
| `prop_tree1` | `length_tree1` / total internal length of tree 1 |
| `prop_tree2` or `prop_support` | Proportional value for dataset 2 |
| `present_in` | `both`, `tree1_only`, `tree2_only`, or `lento_only` |
| `grouping` | Human-readable split |

Rows are sorted: shared branches first (by tree1 length descending), then
tree1-only, then tree2/lento-only.  Summary counts and totals are printed to
stdout, including the normalization scale factor applied.

Pendant (tip) branches are excluded; only non-trivial internal branches
(both sides of the split contain ≥2 taxa) are reported.

The `--tax-order` file fixes bit positions independently of Newick traversal
order, so the same bipartition strings are produced regardless of how the
root or taxon order differs between the two inputs.

## newick_to_biparts.py

Utility to extract all **non-trivial** bipartitions (both sides of the split contain at 
least two taxa) from a Newick tree and write them as canonical binary strings to stdout.  
Output is suitable for direct use with `-t`/`--tree-biparts`, `-p`/`--plot-biparts`, or 
`-b`/--bipart-order in `partcompat.py` and `sample_loci.py` as well as `--omit-biparts` in
`compare_trees.py` (files used for `--omit-biparts` will typically require editing before use).

### Usage

```
newick_to_biparts.py TREE.nwk [TAX_ORDER.txt]
```

If a taxon order file is supplied, bit positions follow that order (matching the `--tax-order` 
file used with `partcompat.py`).  If omitted, taxa are ordered by first appearance in the Newick string.

The tree may be rooted or unrooted; the root position does not affect the bipartitions extracted 
because the taxon order file fixes bit positions independently of traversal order.

Branch lengths and support values are stripped.  Bracketed comments
(e.g. `[&R]`) are removed.

### Obtaining Jakobsen hexadecimal bipartition indices

`newick_to_biparts.py` will output the Jakobsen hex codes for each bipartition as comments if `--hex` 
is passed. 

```bash
python3 newick_to_biparts.py tree.nwk taxa.txt --hex >  hex_annot.biparts
```

The newick tree and optional taxon list are positional arguments (the treefile must be passed 
before the taxon list) but `--hex` can be passed in any position. The hex codes are reported on 
comment lines so they will be ignored 

### Combining multiple trees

Because `partcompat.py` deduplicates entries in bipartition files automatically, outputs from 
multiple `newick_to_biparts.py` calls can be concatenated directly:

```bash
python3 newick_to_biparts.py tree1.nwk taxa.txt >  combined.biparts
python3 newick_to_biparts.py tree2.nwk taxa.txt >> combined.biparts
python3 newick_to_biparts.py tree3.nwk taxa.txt >> combined.biparts

python3 partcompat.py -i alignment.fasta -o results \
    --tax-order taxa.txt \
    -p combined.biparts
```

The `#` comment lines at the top of each file are ignored, so the concatenated file remains 
valid without any editing.

## paup_pstats.py

Computes parsimony statistics (CI, RI, RC) for binary character matrices by running PAUP\*.  These statistics are the traditional tree-assumption-dependent measures of homoplasy, complementing the tree-free compatibility score produced by `partcompat.py`.  Because all most-parsimonious trees for a given dataset have identical CI/RI/RC values, `pscores 1` (the first MPT) is used and the trees themselves are discarded.

**Note:** CI, RI, and RC assume a single underlying tree topology and cannot distinguish hemiplasy (deep coalescence causing apparent homoplasy) from true homoplasy.  The compatibility score from `partcompat.py` makes no tree assumption and therefore reveals different information when the data subject to conflict due to incomplete lineage sorting or hybridization as well as true homoplasy.

### Requires

PAUP\* 4.0 (Swofford 2002), installed separately and accessible either on `$PATH` or via `--paup`.

### Usage

```
paup_pstats.py (-f FILE | -i INDIR) -o OUTDIR --paup PATH [options]
```

**Input modes (mutually exclusive):**

| Flag | Description |
|---|---|
| `-f` / `--file` | Single alignment file (relaxed PHYLIP or Nexus, auto-detected) |
| `-i` / `--indir` | `sample_loci.py` output directory; processes all `replicate_*.phy` files |

**Required:**

| Flag | Description |
|---|---|
| `-o` / `--outdir` | Output directory for working Nexus files, scores files, and summary |
| `--paup PATH` | Path to PAUP\* executable (default: `paup`) |

**Search modes (mutually exclusive):**

| Flag | Description |
|---|---|
| *(default)* | Thorough heuristic: `hsearch addseq=random increase=auto` |
| `--bandb` | Branch-and-bound exact search.  Recommended for ≤12 taxa; warns if applied to larger datasets. |
| `--lazy` | Fast heuristic: `hsearch maxtrees=1000 increase=no nchuck=10` |

**Heuristic search options:**

| Flag | Description |
|---|---|
| `--nreps N` | Number of random-addition `hsearch` replicates (default: 100) |
| `--seed N` | Base random seed (auto-generated and reported if omitted) |

**Coding and ordering options** (apply to PAUP\* analysis and, where relevant, to `--compat`):

| Flag | Description |
|---|---|
| `-m` / `--mode MODE` | Site coding mode (default: `RY`; same options as `partcompat.py`).  Only PI sites under this scheme are passed to PAUP\*.  Also passed to `partcompat.py` when `--compat` is used. |
| `--codon-positions` | Restrict to specific codon positions (default: `all`) |
| `--tax-order FILE` | Taxon order file; reorders sequences before processing for consistency with `partcompat.py` outputs |

**Single-file mode options:**

| Flag | Description |
|---|---|
| `--compat` | Also run `partcompat.py` on the input (using the same `--mode` and `--codon-positions`) and include the compatibility score in output |

**Other:**

| Flag | Description |
|---|---|
| `--redo` | Overwrite existing output directory contents |

### How it works

For each input file the program:

1. Reads the alignment (FASTA, PHYLIP, or Nexus; auto-detected) and applies the same mode-aware coding pipeline as `partcompat.py`: nucleotide sequences are recoded to binary under the chosen `--mode`, and only **parsimony-informative sites** under that scheme are retained.  This ensures that CI, RI, and RC are computed on exactly the same site set as the compatibility score, and that invariant characters (which would artificially inflate CI) are excluded.
2. Writes a Nexus data block containing the PI-only binary matrix.
3. Appends a `Begin PAUP;` block containing the chosen search command,
   `pscores 1 / ci ri rc scorefile=… replace;`, `cleartrees;`, and `quit;`.
4. Runs PAUP\* in batch mode (`paup -n`).
5. Parses CI, RI, and RC from the score file written by `pscores`.

In directory mode, compatibility scores are also read from the
`replicate_NNNN.compat.txt` files produced by `sample_loci.py`, allowing direct comparison of all four metrics in a single summary table.

### Outputs

**Single-file mode** — `OUTDIR/STEM.pstats.tsv` with one data row plus
summary footer rows.

**Directory mode:**

```
replicate_NNNN.nex          Nexus file with PAUP* block (working file)
replicate_NNNN.scores.txt   Raw pscores output from PAUP*
pstats_summary.tsv          Per-replicate table with CI, RI, RC, and compat score
pstats_run_info.txt         Seed, search settings, and summary statistics
```

**`pstats_summary.tsv` columns:**

| Column | Description |
|---|---|
| `replicate` | Replicate identifier (or summary statistic label in footer rows) |
| `n_sites` | Number of characters in the binary matrix |
| `tree_length` | Parsimony tree length (steps) |
| `CI` | Consistency index (0–1; higher = less homoplasy) |
| `RI` | Retention index (0–1; higher = more synapomorphies retained) |
| `RC` | Rescaled consistency index (CI × RI) |
| `compat_score` | Compatibility score from `partcompat.py` *(directory mode only)* |

Footer rows in the TSV report mean, SD, min, and max for each metric.

**stdout** — one-line summary per metric, e.g.:
```
CI:    mean=0.7021  SD=0.0312  min=0.6104  max=0.7895  (n=100)
RI:    mean=0.8120  SD=0.0218  min=0.7543  max=0.8634  (n=100)
RC:    mean=0.5703  SD=0.0287  min=0.4892  max=0.6411  (n=100)
Compat: mean=0.6238  SD=0.1434  min=0.4286  max=0.8929  (n=100)
```

### Examples

Single locus with compatibility score:
```bash
python3 paup_pstats.py -f locus.phy -o locus_stats/     --paup /usr/local/bin/paup     --compat --tax-order taxa.txt
```

Genome-wide replicates from `sample_loci.py` output:
```bash
python3 paup_pstats.py -i genome_wide_results/ -o pstats/     --paup paup     --lazy
```

Small dataset with exact search:
```bash
python3 paup_pstats.py -f small.phy -o small_stats/     --paup paup     --bandb
```

Shell loop over individual loci (for comparison to genome-wide):
```bash
mkdir locus_pstats
for f in loci_directory/*.phy; do
    base=$(basename "$f" .phy)
    python3 paup_pstats.py -f "$f" -o "locus_pstats/$base/" \
        --paup paup --compat --tax-order taxa.txt
done
```

## Bipartition file format

Several programs in the suite (`partcompat.py`, `sample_loci.py`, `analyze_loci.py`, `plot_lento.py`, `compare_trees.py`, and `id_compat.py`) use the same plain-text bipartition file format for `-t`/`--tree-biparts`, `-p`/`--plot-biparts`, `-b`/`--bipart-order`, `--omit-biparts`, `--bipart-file`, and `--bipart`:

- One bipartition per line.
- Each line is either:
  - A **binary string** of length equal to the number of taxa (e.g. `1110000`). The canonical form has the first character `1`; if you supply a string starting with `0` it is automatically flipped.
  - A **Jakobsen hexadecimal string** (e.g. `6`, `0x6`, `6A`), which is the bitwise complement of the canonical binary bitmask.
- Lines beginning with `#` and blank lines are ignored (comments).
- `--tree-biparts` and `--plot-biparts` deduplicate entries silently.
- `--bipart-order` requires each bipartition to appear exactly once; a duplicate triggers an error with the line numbers of both occurrences.

## Error handling

All programs in the PartiPy suite exit with a clean `ERROR:` message (and no
Python traceback) if a required file or directory is not found.  This covers
input alignments, taxon order files, bipartition files (all formats), tree
files, Lento TSV files, and input directories.

---

## Taxon order file

A plain-text file with one taxon name per line.  This file:

- Sets the left-to-right bit position in all bipartition strings.
- Must be the **same file** passed to both `partcompat.py` and
  `newick_to_biparts.py` for bipartition strings to be directly comparable.
- Is required (not optional) for `sample_loci.py`.
- Is strongly recommended for `partcompat.py` whenever outputs from different alignments are to be compared.

Sequences present in an alignment but absent from the taxon order file are appended at the 
end with a warning.  Names in the taxon order file but absent from a specific alignment are 
padded with missing data (`?`).
